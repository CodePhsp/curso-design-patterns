## Objetivo

Desenvolver aplicações com boa organização, clareza arquitetural e excelência técnica.

## Design Patterns

### Uma breve contextualização

Os designs patterns fazem parte da disciplina de *Projeto de Software* - Engenharia de software. Os padrões visam aplicar soluções reutilizáveis para problemas recorrentes no desenvolvimento de software orientado a objetos, cujo objetivo é desenvolver código organizado, reutilizável e de fácil manutenção.

### A divisão dos padrões

Os padrões são dividios em 3 categorias, sendo elas: a criacional, a estrutual e a comportamental. Cada uma possui propósitos distintos.

Os padrões criacionais contolam e abstraem o processo de criação de objetos. Já os padrões estruturais organizam e compõem classes e objetos, enquanto os padrões comportamentais definem como os objetos devem interagir.

A utilização desses padrões reforça a aplicação dos princípios SOLID. Além disso, alguns padrões possuem variações. O padrão Proxy pode ser implementado como proxy virtual, proxy remoto, proxy de proteção e prox inteligente (smart proxy).

## Estrutura do repositório

```text
curso-design-patterns/
├── criacional
|    ├── nome-do-padrao/
|    ├── README.md
|    ├── desafio.md
|    ├── implementacao.py
|    ├── ....py
|    └── fluxo.png
├── estrutural
└── comportamental
   
```

- **README.md**: explicação teórica do padrão;
- **desafio.md**: problema proposto;
- **implementação**: solução desenvolvida;
- **fluxo.png**: fluxo de execução.
